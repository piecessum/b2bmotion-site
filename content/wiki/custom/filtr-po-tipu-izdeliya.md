Если вы хотите настроить работу фильтра по Типу изделия, то вам надо передавать нам из 1с в шт свойства для этого фильтра в определенном порядке

![](/wiki/custom/filtr-po-tipu-izdeliya/01.webp)

{  
"features": \[  
{  
"unit": {  
"name": "В",  
"id": "EU570079"  
}  
},  
{  
"name": "Номин. напряжение перемен. тока (AC)",  
"id": "EF004220",  
"type": "N",  
"sort": 12,  
"value": {  
"number": 230  
}  
},  
{  
"name": "Удаленный сигнал",  
"id": "EF004192",  
"type": "L",  
"sort": 11,  
"value": {  
"boolean": false  
}  
},  
{  
"name": "Конструктивный размер (габарит)",  
"id": "EF002566",  
"type": "A",  
"sort": 10,  
"value": {  
"string": "1 модуль",  
"id": "EV006931"  
}  
}  
\],  
"class": {  
"synonyms": "Combined arrester for power supply systems",  
"name": "Комбинированный разрядник для систем электроснабжения",  
"id": "EC001457",  
"name\_en": "Combined arrester for power supply systems"  
}  
}  
  
Для сортировки (порядок выдачи свойств) нужно указывать поле сортировки в свойствах (sort)  
  
Поле features - массив свойств для фильтра. Они могут быть 3 типов: A, L, N. У каждого типа своя структура описания свойства.  
  
  
Поле class - описание типа изделия

> [!CALLOUT color=#4a9f2d]
> Свойства "Тип изделий" сортируются либо по количеству товаров, либо по релевантности поисковой выдачи
