Любой из ваших клиентов может получить доступ к информации, хранящейся в вашей [B2B-системе](/) через API.  
Сценарии работы:  

-   Интеграция B2B-системы с интернет-магазином клиента. В этом случае вы выступаете как поставщик, а ваш клиент забирает к себе ваш каталог. На своей стороне он может устанавливать свою наценку и продавать через свой интернет-магазин;
-   Интеграция B2B-системы с 1С клиента. Подходит для частых и крупных покупателей, которым удобно делать заказы из своей 1С, не авторизуясь при этом в вашей B2B-системе

  
Для интеграции вашему клиенту выдается уникальный API-ключ и доступ к документации API:  
[https://export-enterprise.b2b.api.docs.3davinci.ru/](https://export-enterprise.b2b.api.docs.3davinci.ru/)  
[https://admin-enterprise.b2b.api.docs.3davinci.ru/#/](https://admin-enterprise.b2b.api.docs.3davinci.ru/#/)  
  
Настройку на своей стороне он осуществляет самостоятельно (силами своих программистов) с помощью этой документации.  
  
В документации написаны запросы, к ним в качестве доменной части адреса дописывается url B2B-системы.

#### Где искать API-ключ?

_Админка - Пользователи - Конкретный пользователь - Просмотр (иконка “глазик”) - API токен_

![](/wiki/tech/podklyuchenie-po-api-dostup-k-b2b-sisteme-dlya-vashikh-klientov/01.webp)

> [!CALLOUT color=#d30000]
> Для каждого клиента API ключ уникален, генерируется при создании пользователя.

Ключ переодически меняется, необходимо заново получать обновленный токен. Необходимо настроить приложение так, что если на любой запрос к B2B в ответ 401 ошибка, оно должно выполнить запрос авторизации. В ответе на этот запрос приходит новый токен, который приложение должно записать в свои настройки.  
  
​Если внести такие правки в приложение сложно, то можно вручную отправить запрос авторизации (например в postman), или посмотреть его в браузере авторизовавшись в B2B-системе.
